# Harness 模板库

> **用途**：讲师上午讲解和下午小组实战参考
> **核心**：提供可直接复制使用的约束模板，降低MBA设计Harness的门槛

---

## 一、Harness 的本质

**定义**：Harness 是一套规则系统，用于引导 Agent 行为、防止幻觉、确保输出符合预期。

**类比**：
> "Harness 就像公司的 SOP（标准作业程序）+ 风控红线。Agent 能力很强，但如果没有 Harness，它就像一辆没有刹车的跑车——速度快，但容易失控。"

**Harness 不是枷锁，而是安全带**：好的 Harness 让 Agent 更快更准，不是更慢更死。

---

## 二、Harness 的四大类型

### 类型1：输出格式约束

**作用**：告诉 Agent"我要什么形式"，而不是让它自由发挥。

#### 模板：财务数据对比表

```
【格式要求】
1. 财务数据用 Markdown 表格呈现，列：指标名称 | 本期值 | 同比变化 | 环比变化 | 说明
2. 核心洞察用 bullet points，每条不超过40字
3. 风险因素单独成段，用⚠️符号标注
4. 最后附一段150字以内的执行摘要

【示例】
| 指标 | 2024Q3 | 同比 | 环比 | 说明 |
|------|--------|------|------|------|
| 营业收入 | 1000亿 | +20% | +5% | 主营业务增长稳健 |
```

#### 模板：调研报告结构

```
【格式要求】
1. 执行摘要（300字以内，结论先行）
2. 市场概览（表格呈现关键数据）
3. 竞争格局（bullet points，主要玩家及份额）
4. 政策环境（表格：国家/补贴/关税/本地化要求）
5. 进入壁垒（每项1-5星评分 + 一句话说明）
6. 初步建议（3条，每条不超过50字）
7. 信息缺口清单（决策需要但本报告缺少的信息）
```

#### 模板：会议纪要结构

```
【格式要求】
1. 会议基本信息（时间、地点、参会人、主题）
2. 各方核心立场（bullet points，每条不超过40字）
   - 甲方立场
   - 乙方立场
3. 已达成共识（编号列出）
4. 待决事项（编号列出，标注争议焦点）
5. Action Items（表格：事项 | 负责人 | 截止时间 | 优先级）
6. 风险提示（模棱两可的表述、可能引发争议的问题）
```

### 类型2：分析过程约束

**作用**：要求 Agent 先给分析框架、依据来源、置信度和不确定性，便于人工审计。

#### 模板：分析框架先行

```
【分析过程要求】
在给出最终结论前，请按以下步骤执行：

步骤1：列出分析框架
"我将从以下X个维度进行分析：..."

步骤2：逐条填充数据
"维度1：...（数据支撑：...）"
"维度2：...（数据支撑：...）"

步骤3：交叉验证
"以下数据点之间存在矛盾：..."
"以下推断的置信度较低：..."

步骤4：得出结论
"基于以上分析，核心结论是：..."
```

#### 模板：依据与置信度可视化

```
【分析过程要求】
对每个核心结论，请按以下格式呈现：

【结论】：（一句话结论）
【依据】：（支撑这个结论的数据/事实）
【逻辑】：（从依据到结论的关键判断）
【置信度】：（高/中/低，并说明原因）
【反方观点】：（如果这个结论有误，最可能的原因是什么）
```

#### 模板：多方案对比

```
【分析过程要求】
对于每个关键决策点，请提供：

方案A：（保守方案）
- 优点：...
- 缺点：...
- 适用条件：...

方案B：（激进方案）
- 优点：...
- 缺点：...
- 适用条件：...

方案C：（折中方案）
- 优点：...
- 缺点：...
- 适用条件：...

【建议】：基于当前信息，我最推荐方案X，原因是...
```

### 类型3：边界约束

**作用**：明确告诉 Agent"你不能做什么"，防止幻觉和越界。

#### 模板：信息来源限定

```
【边界约束】
- 只基于上传的材料进行分析，不要补充训练数据中的信息
- 如果某项信息在材料中没有提及，标注[材料中未提及]而不是编造
- 如果材料之间存在矛盾，列出矛盾点并标注[需核实]
- 禁止使用2024年10月之后的信息（除非材料中明确包含）
```

#### 模板：禁止行为清单

```
【边界约束】
- 禁止编造未公开的财务数据、客户名单、合作协议
- 禁止给出具体的投资建议（买入/卖出/持有/目标价）
- 禁止使用"肯定""必然""绝对"等确定性表述
- 禁止基于单一数据点得出全局结论
- 禁止忽略反面证据（如果存在与主流观点矛盾的数据，必须提及）
```

#### 模板：角色边界

```
【边界约束】
你的角色是分析师，不是决策者。
- 你可以提供分析、数据、框架
- 你不可以代替投资委员会做出决策
- 你不可以预测股价或市场走势
- 你不可以推荐具体的交易策略
```

---

### 类型4：校验机制

**作用**：让 Agent 自我检查，发现自身可能的错误。

#### 模板：数据异常检测

```
【校验机制】
完成分析后，请执行以下自检：

1. 数据合理性检查
   - 毛利率是否在0%-100%之间？
   - 同比增长率是否在合理范围内（如不超过±200%）？
   - 现金流和净利润的方向是否一致？
   - 如有异常，标注[数据异常，建议核实]

2. 逻辑一致性检查
   - 各项数据之间是否存在矛盾？
   - 推断是否基于充分的数据支撑？
   - 结论是否过度泛化？

3. 信息完整性检查
   - 是否遗漏了材料中的关键信息？
   - 是否忽略了反面证据？
```

#### 模板：不确定性声明

```
【校验机制】
完成分析后，请明确回答以下问题：

1. "以上分析中，我最不确定的数据点是..."（列出3个）
2. "如果我的分析有误，最可能错在..."（列出2个潜在错误点）
3. "为了提升分析的可靠性，我还需要..."（列出3个需要补充的信息）
```

#### 模板：版本对比

```
【校验机制】
如果你在某个问题上有多个可能的答案，请：

1. 列出所有可能的解释
2. 对每个解释给出支持证据和反对证据
3. 评估每个解释的置信度（高/中/低）
4. 说明你选择当前答案的理由
5. 如果你不确定，明确说"我不确定"而不是猜测
```

## 三、场景化 Harness 模板

### 模板A：财报分析检查清单

```
【Harness：财报分析】

## 输出格式
1. 核心指标表格（列：指标 | 本期 | 同比 | 环比 | 行业均值 | 说明）
2. 积极变化（2条，bullet points，每条不超过40字）
3. 关注风险（2条，bullet points，每条不超过40字）
4. 投资价值摘要（150字以内）

## 边界约束
- 只基于上传的财报PDF，不补充外部信息
- 如某项指标PDF未披露，标注[PDF未披露]
- 禁止给出买入/卖出建议

## 校验机制
- 完成后列出3个最不确定的数据点
- 如发现同比/环比数据异常（>±50%），标注[数据异常]
- 用一句话总结："以上分析中，我最可能遗漏的是..."
```

### 模板B：商业模式画布填充规范

```
【Harness：商业模式画布】

## 输出格式
按 BMC 9模块逐一呈现：
1. 客户细分
2. 价值主张
3. 渠道通路
4. 客户关系
5. 收入来源
6. 核心资源
7. 关键业务
8. 重要合作
9. 成本结构

每个模块：
- 至少2个关键要点
- 可确认的标注[公开信息]
- 推断的标注[推断]
- 信息不足的标注[信息不足]
- 最后附200字商业模式总结

## 边界约束
- 禁止编造具体财务数据（收入、利润率、成本）
- 禁止编造未公开的合作关系或客户名单
- 禁止过度乐观或过度悲观的表述

## 校验机制
- 完成后列出2个最大优势和2个最大风险
- 如某个模块完全无法填充，说明原因
- 用一句话总结："该商业模式最脆弱的一环是..."
```

### 模板C：会议纪要整理SOP

```
【Harness：会议纪要】

## 输出格式
1. 会议基本信息
2. 各方核心立场（甲方/乙方，bullet points，每条≤40字）
3. 已达成共识（编号）
4. 待决事项（编号，标注争议焦点）
5. Action Items（表格：事项 | 负责人 | 截止时间 | 优先级）
6. 风险提示

## 边界约束
- 只基于文字稿，不补充未提及信息
- 说话人不明时标注[说话人不明]
- 语义不明时保留原文并标注[需确认]
- 禁止添加主观判断

## 校验机制
- 完成后列出3个最可能引发后续争议的问题
- 检查Action Items是否覆盖了所有待决事项
- 用一句话总结："本次会议的隐性风险是..."
```

### 模板D：竞品对比矩阵

```
【Harness：竞品对比】

## 输出格式
1. 对比维度（由用户指定，如功能、价格、性能、服务）
2. Markdown表格（行：产品/公司，列：对比维度）
3. 每项评分（1-5星，附一句话说明）
4. 综合排名（附一句话理由）
5. 差异化洞察（3条，每条不超过50字）

## 边界约束
- 只基于公开可验证的信息
- 信息未公开时标注[未公开]而不是猜测
- 禁止引用无法验证的用户评价或传闻

## 校验机制
- 完成后列出2个对比维度中信息最不充分的
- 检查是否存在" apples to oranges "的比较（不同定位的产品强行对比）
- 用一句话总结："这个对比中，最大的盲区是..."
```

### 模板E：调研报告生成规范

```
【Harness：调研报告】

## 输出格式
1. 执行摘要（300字，结论先行）
2. 市场概览（表格：关键数据）
3. 竞争格局（主要玩家，bullet points）
4. 政策环境（表格：各国政策对比）
5. 进入壁垒（1-5星评分）
6. 初步建议（3条，每条≤50字）
7. 信息缺口清单

## 边界约束
- 每条数据标注来源
- 数据无法确认时标注[待核实]
- 禁止给出具体投资金额或回报预测
- 承认报告局限性："基于公开信息，未包含实地调研"

## 校验机制
- 完成后列出3个最关键的信息缺口
- 检查数据来源是否多样化（至少3个不同来源）
- 用一句话总结："如果决策者只能记住一个结论，应该是..."
```

### 模板F：IC Memo 生成规范

```
【Harness：IC Memo】

## 输出格式
1. 项目概述（200字：公司、行业、轮次、金额）
2. 投资亮点（3条，bullet points，每条≤50字）
3. 关键风险（3条，bullet points，每条≤50字）
4. 财务分析（表格：核心指标及预测）
5. 团队评估（bullet points）
6. 市场机会（TAM/SAM/SOM，表格）
7. 后续尽调判断（不给出投资决策，只给出"值得深入核验"或"需先补充信息"）

## 边界约束
- 只基于提供的材料，不补充外部信息
- 预测数据标注[预测，基于管理层指引]
- 禁止给出具体估值或出价建议
- 对尽调中发现的问题如实披露

## 校验机制
- 完成后列出2个最可能推翻当前结论的风险因素
- 检查是否存在"确认偏误"（只收集支持投资的证据）
- 用一句话总结："如果我是投资委员会，我最担心的是..."
```

### 模板G：政策影响评估

```
【Harness：政策影响评估】

## 输出格式
1. 政策要点摘要（200字，核心条款）
2. 直接影响（对目标行业/企业的即时影响，bullet points）
3. 间接影响（对上下游、竞争格局的中长期影响）
4. 机会分析（2条，政策可能带来的新机会）
5. 风险分析（2条，政策可能带来的新风险）
6. 应对建议（3条，每条≤50字）
7. 不确定性说明（政策执行细则尚未明确的部分）

## 边界约束
- 只基于已发布的政策文本，不猜测未发布的细则
- 对政策解读存在分歧的地方，列出不同观点
- 禁止过度解读（如把"鼓励"说成"强制"）

## 校验机制
- 完成后列出2个政策执行中最大的不确定性
- 检查是否忽略了政策的过渡期安排
- 用一句话总结："该政策最可能被市场低估的影响是..."
```

## 四、通用防幻觉约束模板

以下约束可适用于任何场景，建议作为"基础Harness"附加到所有Prompt中：

```
【基础防幻觉约束】

1. 信息来源标注
   - 每条事实性陈述必须标注信息来源
   - 来源格式：[来源类型：具体出处]
   - 示例：[来源：公司2024年Q3财报，第15页]

2. 不确定性处理
   - 不确定的信息，明确说"不确定"或"信息不足"
   - 禁止用猜测填充空白
   - 推断的内容标注[推断：基于...的合理推测]

3. 数据合理性检查
   - 检查数据是否在合理范围内
   - 发现异常数据标注[数据异常]
   - 矛盾的数据列出矛盾点

4. 反面证据呈现
   - 不要只呈现支持你结论的证据
   - 主动列出反面证据和替代解释
   - 承认分析的局限性

5. 自我怀疑机制
   - 完成后列出3个你最不确定的点
   - 列出2个你的分析可能出错的原因
   - 列出3个需要补充的信息

6. 禁止行为
   - 禁止编造未公开的数据、合作、客户
   - 禁止使用确定性表述（"必然""绝对""肯定"）
   - 禁止基于单一来源得出全局结论
   - 禁止忽略时间因素（用2023年数据推断2025年趋势）
```

## 五、课堂常用 Harness 组合包

### 组合包1：上午跟练最小版

适合 5 分钟跟练，不追求完整，只让学员体验 Harness 的效果。

```text
【最小 Harness】
1. 只基于我提供的材料回答，不补充外部信息。
2. 材料未提及的信息，请标注[材料未提及]。
3. 输出最后写一句："我最不确定的部分是..."
```

### 组合包2：管理层汇报版

适合市场调研、竞品分析、经营分析等汇报类任务。

```text
【Harness：管理层汇报】

## 格式
- 先给 5 行以内执行摘要
- 再给结构化表格
- 最后给风险、信息缺口和下一步建议

## 边界
- 不使用没有来源的精确数字
- 不把推断写成事实
- 不用"必然""确定""全面领先"等强判断词

## 可审计
- 每条关键结论标注依据
- 每条建议说明前提条件
- 输出最后列出 3 个需要人工复核的问题
```

### 组合包3：高风险分析版

适合投资、财务、政策、法务、重大经营决策等场景。

```text
【Harness：高风险分析】

## 来源限定
- 只基于给定材料分析
- 外部信息不得使用，除非用户明确要求并提供来源
- 材料之间矛盾时，列出矛盾点，不自行裁决

## 角色边界
- 你是分析助理，不是最终决策者
- 可以提供分析框架、风险、信息缺口
- 不可以给出买入/卖出/持有、目标价、法律结论或最终经营决策

## 输出要求
- 结论 | 依据 | 置信度 | 反方证据 | 需核实问题
- 对每个结论给出高/中/低置信度
- 低置信度结论不得放入执行摘要

## 自检
- 列出最可能错的 3 个地方
- 列出最需要人工复核的 3 个事实
- 列出材料不足导致不能判断的内容
```

### 组合包4：长任务分阶段版

适合报告生成、商业计划书、行业研究等复杂任务。

```text
【Harness：分阶段执行】

不要一次生成最终稿。请按阶段执行：

阶段1：材料事实提取
- 只提取事实，不做判断
- 输出材料事实表，并标注来源

阶段2：分析框架设计
- 提出分析维度
- 说明每个维度为什么重要

阶段3：初稿生成
- 按框架填充内容
- 每个核心结论标注依据

阶段4：自检与修订
- 查找无依据表述
- 列出反方证据
- 列出信息缺口

当前只执行阶段1。完成后等待我确认，再进入下一阶段。
```

## 六、Harness 诊断表：看到输出后怎么补规则

| Agent 输出问题 | 说明缺了什么 Harness | 下一轮可加的规则 |
|----------------|----------------------|------------------|
| 输出很长但没有重点 | 缺格式约束 | "先给 5 行摘要，再给表格；每条不超过 40 字" |
| 数据没有来源 | 缺来源约束 | "每个数字后标注来源；无法定位则写[需人工核实]" |
| 编造材料中没有的信息 | 缺边界约束 | "只基于给定材料；未提及则标注[材料未提及]" |
| 只写利好不写风险 | 缺反面证据 | "每个结论必须列一个可能反方解释" |
| 直接给决策建议 | 缺角色边界 | "你提供分析，不替代管理层/投委会决策" |
| 推理跳跃 | 缺过程约束 | "按'依据 -> 逻辑 -> 结论'三列输出" |
| 不承认不确定 | 缺自检机制 | "最后列出最不确定的 3 个点" |
| 多轮后开始跑偏 | 缺目标重置 | "回到原任务：...；忽略无关讨论" |

课堂用法：

1. 先让 Agent 犯错。
2. 让学员判断属于表格哪一类问题。
3. 只加一条 Harness 重新跑。
4. 对比修改前后的输出差异。

## 七、不同风险等级的 Harness 强度

| 风险等级 | 场景 | Harness 强度 | 最少要求 |
|----------|------|--------------|----------|
| 低风险 | 邮件润色、标题改写、头脑风暴 | 轻量 | 不新增事实、保留原意 |
| 中风险 | 市场调研、竞品对比、会议纪要 | 中等 | 来源标注、格式约束、信息缺口 |
| 高风险 | 财报、投资、政策、法务、人事 | 强 | 来源限定、角色边界、自检、人工复核 |

高风险任务建议使用这句话开头：

```text
这是一个高风险分析任务。请优先保证可追溯和不越界，而不是追求结论完整。
```

## 八、Harness 设计原则

### 原则1：具体 > 抽象

```
❌ "请准确分析"
✅ "每个数据点标注来源，不确定时标注[待核实]"
```

### 原则2：可执行 > 完美

```
❌ 设计20条约束，Agent 难以全部遵守
✅ 设计5-8条核心约束，确保Agent能执行到位
```

### 原则3：迭代 > 一步到位

```
第一轮：基础约束 → 观察输出 → 发现问题
第二轮：增加针对性约束 → 观察输出 → 发现问题
第三轮：微调约束 → 达到可用质量
```

### 原则4：约束 + 引导 = 最佳

```
Harness 不是只说"不能做什么"，也要说"建议怎么做"。

❌ "禁止编造数据"
✅ "禁止编造数据；如果信息不足，建议使用'信息不足，建议补充...'的表述"
```

## 九、Harness 效果评估

如何判断你的 Harness 是否有效？

| 评估维度 | 有效Harness的表现 | 无效Harness的表现 |
|---------|----------------|----------------|
| **输出一致性** | 多次执行同一任务，输出结构稳定 | 每次输出格式都不同 |
| **幻觉率** | 编造数据的情况显著减少 | 仍然频繁编造 |
| **可审计性** | 能追溯每个结论的来源和推理 | 结论像"黑箱"，无法验证 |
| **人工干预频率** | 需要人工纠正的次数减少 | 每次都需要大量人工修正 |
| **输出可用度** | 直接可用或仅需微调 | 需要重写或大量修改 |

## 十一、一句话总结

> **Harness 的本质是"用规则换质量"。你投入时间设计好的约束，会在每一次 Agent 执行中回报你——更准、更稳、更可信。**
