# Prompt 设计对比材料

> **用途**：讲师上午讲解和下午小组实战参考
> **核心**：展示"入门Prompt"与"专家Prompt"的差异，帮助MBA建立结构化Prompt思维

---

## 一、Prompt 设计原则：CO-STAR 简化框架

| 要素 | 含义 | 一句话说明 |
|------|------|-----------|
| **C**ontext | 背景 | "我们在什么情境下做这件事？" |
| **O**bjective | 目标 | "具体要完成什么任务？" |
| **S**tyle | 风格 | "用什么语气、什么专业程度说话？" |
| **T**one | 语调 | "情感色彩是什么？冷静？热情？审慎？" |
| **A**udience | 受众 | "给谁看？投委会？客户？下属？" |
| **R**esponse | 格式 | "输出什么形式？表格？PPT大纲？报告？" |

**第六要素（建议补充）**：

| 要素 | 含义 | 一句话说明 |
|------|------|-----------|
| **Constraint** | 约束 | "不能做什么？不确定时怎么办？" |

## 二、各场景 Prompt 对比

### 场景：市场调研

#### 入门 Prompt（❌）

```
帮我调研一下全球商学院的AI课程。
```

**典型问题**：
- 目标模糊："调研"是什么意思？要对比？要总结？
- 范围无限："全球"太大，"商学院"太泛
- 没有格式要求：可能输出一大段文字，无法直接使用
- 没有约束：Agent 可能编造不存在的课程

**预期输出**：泛泛而谈的 Wikipedia 式介绍，无法用于课程设计决策。

---

#### 专家 Prompt（✅）

```
【Context】
中欧商学院计划开设一门面向企业高管的"AI与产业变革"短期课程，时长2天，目标学员为VP及以上级别。

【Objective】
调研全球5家顶尖商学院（哈佛商学院、斯坦福GSB、INSEAD、沃顿商学院、MIT斯隆）的同类课程设置，为中欧的课程差异化定位提供依据。

【Style】
资深教育咨询顾问风格，数据驱动，避免主观评价。

【Tone】
客观、冷静、略带审慎。

【Audience】
中欧商学院课程设计委员会，他们熟悉商学教育但不一定了解AI技术细节。

【Response】
1. Markdown表格呈现对比（列：商学院、课程名称、时长、核心模块、目标学员、公开价格、信息来源）
2. 基于对比提炼3条差异化建议（中欧可以突出的独特价值）
3. 每条建议不超过50字

【Constraint】
- 优先使用各商学院官网2023-2025年的公开信息
- 如果某家商学院没有直接对应的课程，标注[无直接对应课程]
- 如果价格信息未公开，标注[价格未公开]
- 不要编造课程价格、学员人数或教授名单
- 完成分析后，请用一句话总结："以上信息中，我最不确定的部分是..."
```

**关键改进点**：

| 改进项 | 入门版 | 专家版 |
|--------|--------|--------|
| Context | 无 | 明确了中欧的定位和目标学员 |
| Objective | "调研" | 具体任务 + 5家目标学校 |
| Style | 无 | 教育咨询顾问，数据驱动 |
| Audience | 无 | 课程设计委员会 |
| Response | 无 | 表格 + 3条建议 + 字数限制 |
| Constraint | 无 | 5条约束 + 自我检查 |

### 场景：财报解读

#### 入门 Prompt（❌）

```
分析一下宁德时代的财报。
```

**典型问题**：
- 没有指定财报期间（2023？2024？Q3？年报？）
- "分析"太泛：要财务指标？要投资亮点？要风险提示？
- 没有格式要求：可能输出大段叙述，无法快速提取关键信息
- 没有约束：Agent 可能基于训练数据补充未披露的信息

---

#### 专家 Prompt（✅）

```
【Context】
我们是某私募基金投研团队，正在评估宁德时代（300750.SZ）2024年Q3的投资价值。已上传宁德时代2024年Q3季报PDF作为分析材料。

【Objective】
基于上传的财报PDF，提取核心财务指标并进行初步分析，为投研团队提供决策参考。

【Style】
资深行业分析师风格，定量为主、定性补充，每个观点必须有数据支撑。

【Tone】
客观、审慎，不夸大积极信号，也不忽视风险因素。

【Audience】
投资委员会成员，他们熟悉财务术语但时间有限，需要在3分钟内抓住核心结论。

【Response】
1. Markdown表格提取以下指标（2024年Q3及同比/环比变化）：
   - 营业收入、归母净利润
   - 毛利率、净利率
   - 经营性现金流净额
   - ROE（净资产收益率）
   - 资产负债率
2. 对每项指标，用一句话说明"这个数字说明什么"
3. 识别2个最积极的变化和2个最需要关注的风险
4. 最后附一段150字的投资价值摘要

【Constraint】
- 只基于上传的PDF内容，不要补充外部信息或训练数据中的信息
- 如果某项指标在PDF中没有明确披露，标注[PDF中未直接披露]
- 不要给出具体的买入/卖出/持有建议，只提供分析
- 如果某项数据看起来异常（如同比变化超过±50%），请标注[数据异常，建议核实]
- 完成后请列出3个你可能遗漏的关键因素
```

**关键改进点**：

| 改进项 | 入门版 | 专家版 |
|--------|--------|--------|
| Context | 无 | 明确基金背景 + 已上传PDF |
| Objective | "分析财报" | 提取指标 + 初步分析 + 决策参考 |
| Style | 无 | 定量为主、数据驱动 |
| Audience | 无 | 投委会，3分钟抓住核心 |
| Response | 无 | 表格 + 要点 + 摘要，明确字数 |
| Constraint | 无 | 5条约束，含防幻觉和自检 |

### 场景：会议纪要整理

#### 入门 Prompt（❌）

```
帮我整理一下这份会议纪要。
```

**典型问题**：
- "整理"太泛：要提取结论？要生成行动项？要识别争议点？
- 没有说明会议类型：是决策会？脑暴会？谈判？不同会议的整理重点完全不同
- 没有格式要求

---

#### 专家 Prompt（✅）

```
【Context】
这是一份某创业公司A轮投资谈判的会议录音转文字稿。参会方：创业公司CEO、COO；投资方合伙人、投资经理。

【Objective】
将原始文字稿整理为结构化的会议纪要，便于双方团队后续跟进和复核。

【Style】
专业、中立、不偏袒任何一方，客观记录各方立场。

【Tone】
冷静、正式，避免口语化表达。

【Audience】
双方团队的核心成员，他们没有参会，需要通过纪要了解全部关键信息。

【Response】
1. 会议基本信息（时间、地点、参会人、会议主题）
2. 各方核心立场（用bullet points，每条不超过40字）
   - 创业方立场
   - 投资方立场
3. 已达成的共识（编号列出）
4. 待决事项（编号列出，标注争议焦点）
5. Action Items（表格：事项、负责人、截止时间、优先级）
6. 风险提示（如果某方立场存在潜在风险，标注）

【Constraint】
- 只基于文字稿内容，不要补充未提及的信息
- 如果某句话的说话人无法确定，标注[说话人不明]
- 对于模棱两可的表述，保留原文并标注[语义不明，需确认]
- 不要添加你的主观判断或建议，只记录客观事实
- 完成后请用一句话总结："本次谈判中，最可能引发后续争议的问题是..."
```

### 场景：商业模式画布（BMC）

#### 入门 Prompt（❌）

```
帮我生成一个商业模式画布。
```

**典型问题**：
- 没有说明是哪个公司
- 没有提供任何信息来源
- Agent 只能基于训练数据，可能过时或不准确

---

#### 专家 Prompt（✅）

```
【Context】
我们正在研究宇树科技（Unitree Robotics）的商业模式，为一份行业分析报告做准备。已上传以下材料作为参考：
- 宇树科技官网产品页截图文字版
- 两篇2024年关于宇树科技的新闻报道
- 一篇机器人行业分析报告（节选）

【Objective】
基于提供的材料，为宇树科技生成一份完整的商业模式画布（Business Model Canvas）。

【Style】
管理咨询顾问风格，结构化、框架化、洞察驱动。

【Tone】
客观、专业，对公司前景既不过度乐观也不刻意贬低。

【Audience】
行业研究报告的读者，包括投资人、行业分析师和企业战略部门。

【Response】
1. 按 BMC 的9个模块逐一呈现（客户细分、价值主张、渠道通路、客户关系、收入来源、核心资源、关键业务、重要合作、成本结构）
2. 每个模块至少列出2个关键要点
3. 对基于公开信息可确认的内容，标注[公开信息]
4. 对需要推断的内容，标注[推断]
5. 对信息不足无法填充的模块，标注[信息不足]
6. 最后附一段200字的商业模式总结

【Constraint】
- 优先基于上传的材料，如果材料中没有相关信息，可以基于常识合理推断，但必须标注[推断]
- 不要编造具体的财务数据（如收入、利润率、成本数字）
- 不要编造未公开披露的合作关系或客户名单
- 如果某个模块你完全无法填充，请说明原因而不是编造内容
- 完成后请列出2个你认为该商业模式的最大优势和2个最大风险
```

### 场景：调研报告生成

#### 入门 Prompt（❌）

```
帮我写一份关于新能源汽车行业的调研报告。
```

**典型问题**：
- "调研报告"太泛：要市场分析？要技术趋势？要竞争格局？要投资建议？
- 没有指定时间范围：2024年现状？2025年预测？
- 没有指定深度：10页摘要？50页深度报告？

---

#### 专家 Prompt（✅）

```
【Context】
我们是某咨询公司产业研究部，正在为一家汽车零部件企业制定进入东南亚市场的战略。需要一份关于"东南亚新能源汽车市场"的基础调研报告。

【Objective】
生成一份结构化的调研报告初稿，涵盖市场规模、竞争格局、政策环境、进入壁垒4个维度，为客户决策提供依据。

【Style】
咨询公司研究报告风格，数据驱动、逻辑清晰、图表化呈现。

【Tone】
客观、务实，承认信息局限，不夸大结论的确定性。

【Audience】
客户企业的战略委员会（CEO、CFO、战略VP），他们熟悉汽车行业但对东南亚市场了解有限。

【Response】
1. 执行摘要（300字以内，结论先行）
2. 市场概览（表格：东南亚6国电动车销量、渗透率、增长率，2023-2024）
3. 竞争格局（主要玩家及市场份额，用bullet points）
4. 政策环境（各国补贴政策、关税、本地化要求，表格呈现）
5. 进入壁垒分析（技术、资金、渠道、政策，每项评分1-5星）
6. 初步建议（3条进入策略建议，每条不超过50字）
7. 信息缺口清单（列出报告中缺少但决策需要的关键信息）

【Constraint】
- 明确标注每条数据的来源（如"根据IEA 2024报告"）
- 如果某项数据无法确认，标注[数据待核实]而不是编造
- 不要给出具体的投资金额或回报预测
- 承认报告的局限性："本报告基于公开信息，未包含实地调研和客户访谈"
- 完成后请用一句话总结："如果客户只能记住一个结论，应该是..."
```

### 场景：政策影响评估

#### 入门 Prompt（❌）

```
帮我分析一下这个政策对企业有什么影响。
```

**典型问题**：
- 没有说明企业类型：制造业、金融机构、平台公司受到的影响完全不同
- 没有区分"政策原文"和"解读文章"
- 容易让 Agent 直接给确定性结论，忽略执行细则和地方差异

---

#### 专家 Prompt（✅）

```
【Context】
我们是一家制造业集团的战略部，正在评估某项新政策对未来 12 个月经营计划的影响。
我将提供三类材料：
1. 政策原文节选
2. 行业协会解读
3. 公司当前业务结构简表

【Objective】
请基于给定材料，识别该政策可能影响公司的关键业务环节，并形成一份供管理层讨论的影响评估底稿。

【Style】
政策研究 + 企业战略分析风格，先区分事实、推断和不确定点。

【Tone】
审慎、客观，不夸大政策利好或利空。

【Audience】
集团 CEO、战略 VP 和业务线负责人，他们需要判断是否调整年度经营重点。

【Response】
1. 政策要点表：条款/要求 | 直接影响对象 | 对公司的可能影响 | 材料依据
2. 影响路径分析：从政策要求 -> 业务环节 -> 经营影响，用 3-5 条 bullet points 表示
3. 风险与机会：各列 3 条，每条不超过 50 字
4. 管理层待确认问题：列出 5 个需要内部补充的信息

【Constraint】
- 只基于给定材料，不补充未提供的政策细则
- 如果政策执行口径不明确，标注[执行口径待确认]
- 不给出"必须进入/退出某市场"等直接决策结论
- 将"政策原文明确写明"和"基于材料推断"分开标注
```

**关键改进点**：

| 改进项 | 为什么重要 |
|--------|------------|
| 指定企业类型 | 政策影响必须落到具体业务结构 |
| 区分材料类型 | 原文、解读、企业材料的可信度不同 |
| 要求影响路径 | 防止直接从政策跳到结论 |
| 加执行口径约束 | 政策类材料常有不确定空间 |

### 场景：竞品功能对比

#### 入门 Prompt（❌）

```
帮我对比一下这几个产品。
```

**典型问题**：
- 不知道对比目的：选型、定价、产品规划还是销售话术？
- 不知道哪些维度重要
- Agent 容易补充网上听来的产品功能，混入未核实信息

---

#### 专家 Prompt（✅）

```
【Context】
我们是某 B2B SaaS 产品团队，正在规划下一季度的数据看板功能。
我将提供 3 个竞品官网功能页的文字摘录，以及我们当前产品功能清单。

【Objective】
请基于给定材料，生成一份竞品功能对比矩阵，识别我们产品的差距和可优先补齐的功能方向。

【Style】
产品经理风格，结构化、以用户场景和产品能力为核心。

【Tone】
务实、克制，不做营销式夸张表达。

【Audience】
产品负责人和研发负责人，他们需要据此讨论下一季度 roadmap。

【Response】
1. 功能对比矩阵：功能维度 | 我方 | 竞品 A | 竞品 B | 竞品 C | 证据来源 | 差距判断
2. 用户场景差距：列出 3 个我方可能覆盖不足的场景
3. 优先级建议：按高/中/低列出 5 个候选改进点，并说明依据
4. 信息缺口：列出无法从官网判断、需要访谈或试用验证的问题

【Constraint】
- 只基于给定官网材料和我方清单，不补充外部资料
- 如果某个功能没有明确提及，标注[未提及]，不要推断为"没有"
- 不使用"领先""碾压""完胜"等主观营销词
- 优先级建议只是讨论输入，不代表最终 roadmap 决策
```

### 场景：数据故事化汇报

#### 入门 Prompt（❌）

```
帮我把这些数据讲成一个故事。
```

**典型问题**：
- "故事"容易变成夸张叙事，而不是业务解释
- 没有说明听众是谁、要推动什么行动
- 没有要求保留反常数据和不确定性

---

#### 专家 Prompt（✅）

```
【Context】
我们是某消费品公司的运营团队，准备向管理层汇报最近 6 个月的渠道销售数据。
我将提供一张 CSV 表格，字段包括：月份、区域、渠道、销售额、毛利率、退货率。

【Objective】
请把数据整理成一份 8 分钟汇报的大纲，帮助管理层理解主要变化、可能原因和需要进一步核实的问题。

【Style】
经营分析汇报风格，结论先行，数据支撑，不做过度解释。

【Tone】
清晰、冷静、面向管理讨论。

【Audience】
总经理、销售负责人、财务负责人，他们关心增长质量和渠道风险。

【Response】
1. 一句话主结论
2. 三个关键发现：每个发现包含"数据表现、可能解释、需核实问题"
3. 一页图表建议：说明适合用柱状图/折线图/散点图中的哪一种，以及原因
4. 管理层讨论问题：列出 3 个可用于会议讨论的问题

【Constraint】
- 不要把相关性说成因果关系
- 如果数据不足以解释原因，标注[原因待核实]
- 保留异常值，不要为了故事流畅而忽略
- 所有结论必须能回到表格中的字段
```

## 三、Prompt 拆分工作流

复杂任务不要指望一个 Prompt 一步到位。建议把任务拆成 4 轮，每轮只做一件事：

| 轮次 | 目标 | Prompt 重点 | 产物 |
|------|------|-------------|------|
| 第 1 轮 | 读材料 | "请只提取事实，不分析" | 材料事实表 |
| 第 2 轮 | 建框架 | "请提出分析维度和判断标准" | 分析框架 |
| 第 3 轮 | 填内容 | "请按框架填充，并标注来源" | 初稿 |
| 第 4 轮 | 自检修正 | "请找出不确定点、缺口和反证" | 可复核版本 |

可复制模板：

```text
我们分四步完成，不要一次给最终报告。

第 1 步：只提取材料事实，输出表格，不做判断。
第 2 步：基于事实表提出分析框架，说明每个维度为什么重要。
第 3 步：按框架生成初稿，每个结论标注材料依据。
第 4 步：自检，列出不确定点、反方证据和需要人工复核的信息。

当前请先执行第 1 步。
```

## 五、常见 Prompt 设计错误速查表

| 错误类型 | 示例 | 修正方法 |
|---------|------|---------|
| **目标模糊** | "分析一下" | 改为"提取...并对比...最后总结..." |
| **范围过大** | "全球行业" | 限定为"3-5个特定国家/公司" |
| **缺少 Audience** | 无 | 加上"面向...他们熟悉...但时间有限" |
| **缺少格式** | 无 | 指定"表格 + bullet points + 摘要" |
| **约束不足** | 无 | 加上"只基于给定材料"、"不确定时标注" |
| **信息过载** | 上传100页材料 | 先人工筛选5页关键内容 |
| **术语假设** | 使用行业黑话 | 在 Context 中解释关键术语 |
| **期望一步到位** | 一次Prompt解决所有问题 | 拆成多轮对话，每轮聚焦一个子任务 |
| **把假设当事实** | "这个市场正在爆发" | 改为"我的假设是市场在增长，请帮我找支持和反证" |
| **让 Agent 替代决策** | "我们应该投哪家公司" | 改为"列出对比维度、风险和信息缺口，不给投资建议" |
| **没有评价标准** | "写得好一点" | 改为"适合 3 分钟阅读，结论先行，每条有依据" |
| **混合多个受众** | "给老板和客户都能看" | 拆成管理层版、客户版两种输出 |

## 六、Prompt 优化提问库

当学员不知道如何改 Prompt 时，讲师可以逐个追问：

| 追问 | 目的 |
|------|------|
| "这个输出最终给谁看？" | 补 Audience |
| "读者看完要做什么动作？" | 补 Objective |
| "你希望它像报告、表格还是邮件？" | 补 Response |
| "哪些内容绝对不能编？" | 补 Constraint |
| "哪些信息来自材料，哪些是你的判断？" | 区分 Context 与假设 |
| "如果只能保留一页，你要什么？" | 收敛范围 |
| "怎么判断 Agent 的答案合格？" | 建立验收标准 |

## 七、一句话 Prompt 检查清单

在发送 Prompt 前，问自己6个问题：

1. [ ] **目标清晰吗？** — Agent 知道要完成什么具体任务吗？
2. [ ] **背景充分吗？** — Agent 了解前因后果和约束条件吗？
3. [ ] **受众明确吗？** — Agent 知道输出给谁看、用什么语气吗？
4. [ ] **格式指定了吗？** — Agent 知道输出什么形式（表格/报告/大纲）吗？
5. [ ] **约束到位了吗？** — Agent 知道不能做什么、不确定时怎么办吗？
6. [ ] **可验证吗？** — Agent 的输出有明确标准可以判断是否合格吗？

如果6个都满足，这是一个高质量的 Prompt。
