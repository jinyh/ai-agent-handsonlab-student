# 场景任务卡目录

> **用途**：下午小组实战时，每组领取一张任务卡
> **使用方式**：8组各选一张，先到先得；允许重复但需换不同目标公司/行业
> **材料边界**：所有任务卡均使用教学模拟材料；公司、政策和市场数据以卡片内日期为准，不要求联网更新为最新事实。

---

## ⚠️ 免责声明

**本目录所有任务卡仅供教学使用，具有以下特点：**

1. **模拟数据**：所有财务数据、市场数据、政策内容均为教学模拟，不代表真实情况
2. **公司名称**：涉及的真实公司名称（如宁德时代、比亚迪、特斯拉等）仅用于教学场景设定，所有相关数据均为虚构
3. **非投资建议**：所有任务卡明确禁止 Agent 输出投资建议（买入/卖出/持有/目标价等），仅用于演示 AI Agent 的信息处理能力
4. **时效性**：材料以卡片内标注日期为准，不反映最新市场情况
5. **教学目的**：本材料旨在帮助学员理解 AI Agent 的能力边界、常见错误和约束方法，不构成任何专业建议

**使用本材料即表示您理解并同意上述声明。**

---

## 📂 关于 data 目录

本目录提供的是**教学模拟数据**，设计为封闭式练习：

**为什么不用真实数据？**
- 避免学员花时间验证数据真实性和时效性
- 专注于 Prompt/Context/Harness 方法的学习和迭代
- 确保所有小组在相同数据基础上练习，便于对比和讨论

**为什么不鼓励联网搜索？**
- 90分钟内，找数据会挤压 Prompt 迭代和翻车复盘的时间
- 教学重点是"如何与 Agent 协作"，而非"如何找数据"
- 封闭数据集能更好地展示 Agent 的幻觉、确认偏误等典型问题

**课后如何应用到真实工作？**
- 将今天学到的 Prompt 框架（CO-STAR）应用到真实数据
- 将今天学到的 Harness 约束（标注来源、列出不确定性）应用到实际场景
- 如需使用真实数据练习，建议课后自行尝试，并对比 Agent 在模拟数据 vs 真实数据上的表现差异

## 场景重叠提醒

> S7（政策影响评估）与 S13（调研报告）的核心材料都涉及东南亚新能源汽车市场。如果两组分别选了 S7 和 S13，需确保关注点不同：**S7 侧重政策影响分析**（欧盟关税 + 美国 IRA 对中国电池企业的影响），**S13 侧重市场进入策略**（东南亚各国的市场规模、竞争格局、进入路径）。建议助教协调时提醒。

## 选择规则

1. **先到先得**：每组派代表向助教举手抢选
2. **避免重复**：鼓励不同组选不同场景
3. **允许重复但需换目标**：如果两组选同一编号，必须选择不同的公司/行业/主题
4. **助教协调**：如果某类场景过于集中，助教可建议分散

---

## 成果要求（所有场景通用）

每组汇报时需展示：

1. **最终成果**：Agent 生成的最终输出
2. **Prompt 迭代记录**：至少展示两个版本的 Prompt，说明为什么调整
3. **翻车记录**：Agent 犯过什么错误？怎么发现的？怎么纠正的？
4. **心得总结**：1句话总结今天最大的收获

每张任务卡顶部都有一个 **5分钟速用框**。如果时间紧，先读速用框，再看任务目标、输出要求和 Prompt 框架。

---

## 时间分配建议（90分钟）

| 阶段 | 时长 | 任务 |
|------|------|------|
| 准备 | 0-20min | 讨论并确定 Prompt 初稿；准备 Context 材料 |
| 第一轮 | 20-50min | Agent 执行 + 人工校验 |
| 迭代 | 50-70min | 根据问题调整 Prompt/Harness |
| 整理 | 70-90min | 整理最终成果和汇报材料 |

> **复杂场景提示**：输出项≥7的场景（S6、S7、S11、S13），Agent 单次执行时间较长，建议只做 **1轮深度迭代+校验**，不追求多轮浅迭代。简单场景（S3、S8、S9、S12）则可做 2-3轮迭代。

---

## 注意事项

- **完成比完美重要**：90分钟内不要求完美，要求"可用"
- **鼓励翻车**：翻车是最好的学习材料，不要害怕展示 Agent 的错误
- **记录过程**：记录员的任务是记录迭代过程，不是记录最终成果
- **善用模板**：`materials/` 目录下的 Prompt 设计指南、Harness 模板库可直接参考
