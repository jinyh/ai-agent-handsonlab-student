# S2: IC Memo 草稿生成

> **类型**：金融
> **难度**：★★★★
> **建议时长**：90分钟
> **适合人群**：投资、投行、PE/VC背景，熟悉投资分析框架

> **📌 教学材料声明**：本任务卡使用模拟数据，所有公司名称、财务数据、市场信息均为教学虚构，不构成任何投资建议。

---

## 5分钟速用框

| 项目 | 建议 |
|------|------|
| **适用场景** | PE/VC 项目初筛、投委会讨论材料草稿、尽调问题整理 |

## 背景设定

你是某PE基金的资深投资经理。基金正在评估对**星辰智驾（虚构公司）**的A轮投资，投资金额约5000万元人民币。

星辰智驾是一家专注于**自动驾驶域控制器**的创业公司，成立于2021年，团队来自某头部自动驾驶公司。公司已获得种子轮融资（金额未公开），现有员工65人。

你的任务是：基于提供的有限材料，生成一份IC Memo（Investment Committee Memorandum，投资决策备忘录）草稿，供投资委员会讨论。


---

## 任务目标

生成一份结构化的IC Memo，包含以下模块：

1. **项目概述**（公司、行业、轮次、金额）
2. **投资亮点**（3条）
3. **关键风险**（3条）
4. **团队评估**
5. **市场机会**（TAM/SAM/SOM估算）
6. **后续尽调判断**（给出"值得深入核验"或"需先补充信息"，并说明信息缺口）


---

## 输出要求

### 格式要求

1. **项目概述**（200字以内）
2. **投资亮点**（3条，bullet points，每条不超过50字）
3. **关键风险**（3条，bullet points，每条不超过50字）
4. **团队评估**（bullet points，优势+短板）
5. **市场机会**（TAM/SAM/SOM表格 + 一句话说明）
6. **后续尽调判断**（不超过100字，给出"值得深入核验"或"需先补充信息"的理由）

### 约束要求

- 只基于提供的模拟材料，不补充外部信息
- 预测数据标注[基于管理层指引]
- 禁止给出具体估值或出价建议
- 对尽调中发现的问题如实披露
- 完成后列出2个最可能推翻当前结论的风险因素

## 建议的 Prompt 框架

```
【Context】
我们是某PE基金投资团队，正在评估星辰智驾（自动驾驶域控制器创业公司）
的A轮投资（5,000万元）。已上传BP摘要、行业研究、尽调访谈纪要。

【Objective】
基于提供的材料，生成一份IC Memo草稿，供投资委员会讨论。

【Style】
资深投资经理风格，平衡乐观与审慎，不回避风险。

【Tone】
专业、冷静，承认信息局限。

【Audience】
投资委员会（合伙人级别），他们熟悉投资流程但不一定了解自动驾驶技术细节。

【Response】
1. 项目概述（200字）
2. 投资亮点（3条，≤50字/条）
3. 关键风险（3条，≤50字/条）
4. 团队评估（bullet points）
5. 市场机会（TAM/SAM/SOM表格）
6. 后续尽调判断（≤100字）

【Constraint】
- 只基于提供的材料
- 预测数据标注[基于管理层指引]
- 禁止给出估值或出价建议
- 如实披露尽调中发现的问题
- 完成后列出2个最可能推翻结论的风险因素
```

## 评价要点

| 维度 | 重点 |
|------|------|
| **Prompt设计** | 是否要求了完整的IC Memo结构？是否禁止了估值建议？ |
| **Context管理** | 是否综合使用了三份材料？是否有遗漏的关键信息？ |
| **Harness约束** | 是否有"如实披露问题"的约束？是否有推翻结论的风险提示？ |
| **输出质量** | Memo结构是否专业？风险披露是否充分？ |
| **迭代过程** | 是否发现了材料之间的矛盾？如何处理？ |


---

## 提示与技巧

### 可能的"翻车"点

1. **Agent 可能过于乐观**：只提取亮点，忽略风险（ Confirmation Bias ）
2. **Agent 可能编造估值**：虽然Prompt禁止，但Agent可能"忍不住"给出一个数字
3. **Agent 可能混淆TAM/SAM/SOM**：需要在Prompt中明确定义这三个概念

4. **如果 Prompt 没写 Audience**：Agent 可能用技术黑话写 Memo，投资委员会看不懂
5. **如果只给了BP而没给尽调纪要**：Agent 只能看到公司的正面表述，无法发现"第一大客户占比60%"和"返修率3%"等风险信号
### 如何提升质量

- 在 Prompt 中明确要求"正面和负面信息平衡呈现"
- 增加约束："对尽调中的负面反馈，不得淡化或省略"
- 要求 Agent "从反对投资的角度，列出3个理由"

### 可扩展的方向

- 增加财务模型（简单的收入、毛利率、现金消耗敏感性分析）
- 增加竞争对比（与某A公司的横向对比）
- 增加退出路径分析（IPO/并购可能性）

## 汇报提示

汇报时建议结构：
1. **场景与目标**（30秒）：选了S2 IC Memo，目标是评估星辰智驾A轮投资
2. **Prompt 展示**（1分钟）：重点说明如何用 Constraint 禁止估值建议、如何要求如实披露尽调负面反馈
3. **翻车展示**（1分钟）：Agent 是否只列亮点忽略风险（确认偏误）？是否忍不住给出估值数字？
4. **成果展示**（1分钟）：IC Memo 草稿，重点看风险披露是否充分
5. **一句话总结**（30秒）
