# S9: 会议纪要智能整理

> **类型**：通用
> **难度**：★★
> **建议时长**：90分钟
> **适合人群**：所有背景，最高频的Agent应用场景

> **📌 教学材料声明**：本任务卡使用模拟数据，所有公司名称、财务数据、市场信息均为教学虚构，不构成任何投资建议。

---

## 5分钟速用框

| 项目 | 建议 |
|------|------|
| **适用场景** | 谈判纪要、项目会议纪要、行动项追踪 |
| **推荐 Context** | `data/S9-material.md` 模拟谈判文字稿 |
| **Harness 起点** | 只基于文字稿；不添加主观判断；说话人不明和语义不明必须标注 |
| **最小成果** | 会议基本信息 + 各方立场 + 共识/待决事项 + Action Items + 风险提示 |
| **汇报重点** | 展示 Agent 是否误判责任人，是否把模糊表述写成已决事项 |


---

## 背景设定

你是某公司的项目助理。今天上午，公司参加了一场**商务谈判**，主题是"某供应商合同续签"。

谈判结束后，你需要将**录音转文字稿**整理为结构化的会议纪要，供双方团队后续跟进。


---

## 任务目标

基于提供的模拟文字稿，生成一份**结构化会议纪要**，包含：

1. **会议基本信息**
2. **各方核心立场**
3. **已达成共识**
4. **待决事项**
5. **Action Items**（行动项）
6. **风险提示**


---

## 操作提示

> 将 `data/` 目录下的 `S9-material.md` 复制到你的项目目录中，让 Agent 读取；也可直接粘贴到 Prompt 的 Context 部分。建议只使用议题1-3，议题4可选择性使用。

## 提供的材料

### 材料：模拟谈判文字稿

```
【模拟商务谈判文字稿】

时间：2024年10月22日 上午9:30-11:00
地点：甲方公司会议室
主题：XX供应商合同续签谈判

参会人：
- 甲方：张总（采购总监）、李经理（采购经理）、王工（技术负责人）
- 乙方：刘总（销售总监）、陈经理（客户经理）、赵工（技术支持）


---

【开场】

张总：感谢刘总团队过来。我们今天主要是谈一下明年的合同续签问题。
目前的合同年底到期，我们需要确定新的合作框架。

刘总：好的，张总。我们非常重视和贵公司的合作。
过去三年合作非常愉快，我们也希望能继续深化合作。


---

【议题1：价格】

张总：先说价格。目前的采购单价是120元/件，
我们希望在明年能有一定幅度的下调。市场行情你们也知道的。

刘总：张总，我理解。但今年的原材料成本上涨了不少，
特别是上游的钢材和芯片，我们的成本压力也很大。
如果降价，我们可能很难保证现有的质量标准。

李经理：刘总，我们也了解成本压力。
但我们的需求量和去年相比增加了30%，
按照规模效应，单价应该可以有一定优惠吧？

刘总：这倒是。如果量能保证，我们可以在现有价格基础上
做一个阶梯定价。比如：
- 年采购量<10万件：维持120元
- 10-20万件：115元
- >20万件：110元

张总：这个方案我们可以接受。但有一个条件：
如果年采购量超过20万件，110元要锁定三年，不能随意涨价。

刘总：三年锁定？这个我需要回去和财务商量一下。
目前看，如果我们能签一个三年框架协议，
110元锁定是可以考虑的。

（双方未达成一致，记录为待决事项）


---

【议题2：交付周期】

王工：说一下交付。目前的交付周期是45天，
我们希望在明年缩短到30天。我们的生产线节奏加快了。

赵工：王工，45天已经是我们的极限了。
如果要缩短到30天，我们需要增加库存，这对我们的资金占用很大。

王工：我们理解。但30天是硬要求。
你们可以考虑在我们工厂附近设一个仓库，
这样既能缩短交付，也能降低你们的物流成本。

赵工：设仓库？这个成本谁出？

张总：仓库租金可以谈。如果年采购量超过15万件，
我们可以承担仓库租金的一半。但有一个条件：
库存周转率要达到每月2次以上。

刘总：这个方案可以考虑。但具体的租金和库存管理责任，
我们需要再细化一下。

（双方达成初步共识，需进一步细化）


---

【议题3：质量标准】

王工：关于质量标准，我们希望在现有基础上增加一个抽检环节。
目前的抽检率是5%，我们希望在明年提高到10%。

赵工：10%？这意味着我们的不良品率要控制在更低水平，
对生产流程的要求更高了。

王工：是的。但这对双方都有好处。
如果抽检合格率连续三个月达到99.5%以上，
我们可以将抽检率降回5%。

刘总：这个激励机制不错。我们接受。
但如果在过渡期内出现质量问题，责任如何界定？

张总：如果是因为乙方生产原因导致的质量问题，
乙方承担全部责任，包括退换货和延误赔偿。
如果是甲方设计变更导致的，责任由甲方承担。

刘总：可以。但希望能在合同中明确"设计变更"的定义范围。

（双方达成初步共识，需进一步细化责任界定）


---

【议题4：付款条款】

李经理：付款条款方面，目前我们是货到60天付款。
我们希望调整为货到90天付款。

刘总：90天？这我们的资金压力太大了。
目前的60天已经是行业平均水平了。

张总：刘总，我们也知道这有难度。
但我们的现金流也在收紧。这样吧，
如果年采购量超过20万件，我们可以接受60天付款不变。
如果低于20万件，就按90天付款。

刘总：这个...我需要回去和财务再确认一下。
目前看，如果能保证20万件的量，60天付款没问题。

（双方未达成一致，记录为待决事项）


---

【总结】

张总：好的，今天谈了不少。总结一下：
- 价格：阶梯定价方案，110元锁定三年（待确认）
- 交付：设仓库，租金分担（需细化）
- 质量：抽检率提高到10%，有激励机制（需细化责任界定）
- 付款：60天 vs 90天（待确认）

刘总：好的。我们回去内部讨论一下，
下周三之前给贵方一个正式回复。

张总：没问题。那今天的会议纪要，
请李经理整理后发给大家确认。


---
```

## 输出要求

### 格式要求

1. **会议基本信息**（时间、地点、参会人、主题）

2. **各方核心立场**（甲方/乙方，bullet points，每条≤40字）

3. **已达成共识**（编号列出）

4. **待决事项**（编号列出，标注争议焦点）

5. **Action Items**（表格：事项 | 负责人 | 截止时间 | 优先级）

6. **风险提示**（模棱两可的表述、可能引发争议的问题）

### 约束要求

- 只基于文字稿，不补充未提及信息
- 说话人不明时标注[说话人不明]
- 语义不明时保留原文并标注[需确认]
- 禁止添加主观判断
- 完成后列出3个最可能引发后续争议的问题

## 建议的 Prompt 框架

```
【Context】
这是一份商务谈判的录音转文字稿，主题是供应商合同续签。
参会方：甲方（采购方）、乙方（供应商）。

【Objective】
将文字稿整理为结构化的会议纪要，便于双方后续跟进。

【Style】
专业、中立、不偏袒任何一方。

【Tone】
冷静、正式，避免口语化。

【Audience】
双方团队的核心成员，他们没有参会，需要通过纪要了解全部关键信息。

【Response】
1. 会议基本信息
2. 各方核心立场（甲方/乙方）
3. 已达成共识（编号）
4. 待决事项（编号，标注争议焦点）
5. Action Items（表格）
6. 风险提示

【Constraint】
- 只基于文字稿
- 说话人不明标注[说话人不明]
- 语义不明标注[需确认]
- 禁止添加主观判断
- 完成后列出3个最可能引发后续争议的问题
```

## 评价要点

| 维度 | 重点 |
|------|------|
| **Prompt设计** | 是否要求了Action Items表格？是否定义了中立立场？ |
| **Context管理** | 是否充分使用了文字稿的全部内容？ |
| **Harness约束** | 是否有"不添加主观判断"的约束？是否有争议提示？ |
| **输出质量** | 纪要是否完整？Action Items是否覆盖了所有待决事项？ |
| **迭代过程** | 是否发现了文字稿中的模糊表述？如何处理？ |


---

## 提示与技巧

### 可能的"翻车"点

1. **Agent 可能遗漏"待确认"事项**：把"待确认"误记为"已达成共识"
2. **Agent 可能混淆说话人**：把甲方的话记到乙方头上
3. **Agent 可能添加主观判断**：如"乙方态度强硬"（原文没有）

4. **如果 Prompt 没要求"禁止添加主观判断"**：Agent 可能加入"乙方态度强硬"等原文中没有的形容词
5. **如果文字稿过长（>170行）直接粘贴**：Agent 可能处理不全，建议只用前3个议题
### 如何提升质量

- 要求"严格按原文记录，不要概括或改写"
- 要求"对每个结论标注原文出处（第几段）"
- 增加校验："检查是否有遗漏的待决事项"

### 可扩展的方向

- 增加"情感分析"（各方情绪变化？）
- 增加"谈判策略分析"（各方的让步点和底线）
- 增加"后续跟进建议"（下一轮谈判的建议议程）

## 汇报提示

汇报时建议结构：
1. **场景与目标**（30秒）：选了S9会议纪要，目标是整理商务谈判文字稿
2. **Prompt 展示**（1分钟）：重点说明如何用 Constraint 禁止主观判断、要求标注[需确认]
3. **翻车展示**（1分钟）：Agent 是否把"待确认"误记为"已达成共识"？是否混淆甲方和乙方的发言？
4. **成果展示**（1分钟）：会议纪要 + Action Items表格，重点看待决事项是否完整
5. **一句话总结**（30秒）
