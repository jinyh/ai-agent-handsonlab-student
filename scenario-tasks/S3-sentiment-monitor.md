# S3: 舆情监控简报

> **类型**：金融/通用
> **难度**：★★
> **建议时长**：90分钟
> **适合人群**：所有背景，对某家上市公司感兴趣即可

> **📌 教学材料声明**：本任务卡使用模拟数据，所有公司名称、财务数据、市场信息均为教学虚构，不构成任何投资建议。

---

## 5分钟速用框

| 项目 | 建议 |
|------|------|
| **适用场景** | 投关舆情摘要、管理层简报、媒体/投资者关注点梳理 |
| **推荐 Context** | 选择一个场景材料：`data/S3-scenario-A.md`、`data/S3-scenario-B.md` 或 `data/S3-scenario-C.md` |
| **Harness 起点** | 只基于模拟新闻材料；不给投资建议；不编造新闻；每条信号标注来自第几条材料 |
| **最小成果** | 舆情摘要 + 正面信号 + 负面信号/风险 + 风险评级 + 建议回应 |
| **汇报重点** | 展示风险评级是否有依据，是否把券商观点当成公司事实 |


---

## 背景设定

你是某上市公司（可选：**宁德时代**、**比亚迪**、**贵州茅台**、或**自己选择的公司**）的投资者关系部门分析师。

公司管理层希望了解近期（过去30天）市场对公司的舆情动态，特别是：
- 投资者和媒体的**主要关注点**
- **正面信号**和**负面信号**
- 是否需要**主动回应**某些舆论


---

## 任务目标

基于提供的模拟新闻/公告材料，生成一份**舆情监控简报**，包含：

1. **舆情摘要**（过去30天市场关注的核心议题）
2. **正面信号**（2-3条）
3. **负面信号/风险**（2-3条）
4. **风险评级**（高/中/低，附理由）
5. **建议回应策略**（是否需要主动沟通？）


---


## 操作提示

> 将 `data/` 目录下的 `S3-scenario-A.md`、`S3-scenario-B.md`、`S3-scenario-C.md` 复制到你的项目目录中，让 Agent 读取；也可直接粘贴到 Prompt 的 Context 部分。默认选择场景A（宁德时代）。

## 输出要求

### 格式要求

1. **舆情摘要**（200字以内，概括过去30天核心议题）
2. **正面信号**（2-3条，bullet points，每条≤40字）
3. **负面信号/风险**（2-3条，bullet points，每条≤40字）
4. **风险评级**（高/中/低，附50字以内理由）
5. **建议回应策略**（是否需要主动沟通？如需要，建议什么方式？）

### 约束要求

- 只基于提供的模拟新闻材料
- 不给投资建议（不输出买入/卖出/持有/目标价等建议）
- 对无法确认的信息（如"某欧洲车企"），保留匿名处理
- 不要编造未提及的新闻或数据
- 风险评级需有明确依据

## 建议的 Prompt 框架

```
【Context】
我们是某上市公司的投资者关系部门分析师。
已上传过去30天的模拟新闻/公告材料（共8条）。

【Objective】
基于提供的材料，生成一份舆情监控简报，
帮助管理层了解市场关注点和潜在风险。

【Style】
专业、客观，既不夸大正面信号，也不放大负面信号。

【Tone】
冷静、审慎，承认信息局限。

【Audience】
公司管理层（CFO/董秘级别），他们熟悉公司业务但需要快速了解外部舆情。

【Response】
1. 舆情摘要（200字）
2. 正面信号（2-3条，≤40字/条）
3. 负面信号/风险（2-3条，≤40字/条）
4. 风险评级（高/中/低 + 50字理由）
5. 建议回应策略

【Constraint】
- 只基于提供的材料
- 保留匿名信息（如"某欧洲车企"）
- 不编造未提及的新闻
- 风险评级需有明确依据
```

## 评价要点

| 维度 | 重点 |
|------|------|
| **Prompt设计** | 是否要求了风险评级？是否定义了Audience？ |
| **Context管理** | 是否覆盖了全部8条材料？是否有遗漏？ |
| **Harness约束** | 是否有"不编造新闻"的约束？风险评级是否有依据？ |
| **输出质量** | 摘要是否准确？正面/负面是否平衡？ |
| **迭代过程** | 是否发现了材料之间的矛盾？如何处理？ |


---

## 提示与技巧

### 可能的"翻车"点

1. **Agent 可能过度解读**：把"正在评估"说成"已经决定"
2. **Agent 可能忽略时间线**：把30天前的新闻和昨天的新闻同等权重处理
3. **Agent 可能情绪传染**：被负面新闻带偏，风险评级过高

4. **如果 Prompt 没要求"按时间顺序梳理"**：Agent 可能不分时间先后地罗列新闻，无法看出舆情演变
5. **如果只给正面新闻不给负面新闻**：Agent 的风险评级会偏低，管理层无法看到需要主动回应的舆情
### 如何提升质量

- 要求 Agent "按时间顺序梳理事件，标注每条新闻的日期"
- 要求 "对每条信号标注信息来源（第几条材料）"
- 增加约束："风险评级需基于事实，不要基于情绪"

### 可扩展的方向

- 增加"情感分析"维度（正面/负面/中性占比）
- 增加"传播力评估"（哪条新闻影响最大？）
- 增加"历史对比"（与上个月的舆情对比）

## 汇报提示

汇报时建议结构：
1. **场景与目标**（30秒）：选了S3舆情监控，目标是整理宁德时代（默认选项）过去30天的舆情
2. **Prompt 展示**（1分钟）：重点说明 Audience 和 Constraint（只基于给定材料、保留匿名信息）
3. **翻车展示**（1分钟）：Agent 是否把"正在评估"说成"已经决定"？风险评级是否过高？
4. **成果展示**（1分钟）：舆情简报，重点看正面/负面是否平衡
5. **一句话总结**（30秒）

> **默认选项**：建议选择宁德时代场景A；如需换公司，可在第一轮完成后切换。
