# S6: 商业模式画布生成

> **类型**：产业
> **难度**：★★★
> **建议时长**：90分钟
> **适合人群**：战略、咨询、创业背景，熟悉BMC框架

> **📌 教学材料声明**：本任务卡使用模拟数据，所有公司名称、财务数据、市场信息均为教学虚构，不构成任何投资建议。

---

## 5分钟速用框

| 项目 | 建议 |
|------|------|
| **适用场景** | 商业模式画布、战略分析初稿、创业项目复盘 |
| **推荐 Context** | `data/S6-material-A.md` 公司公开信息 + `data/S6-material-B.md` 行业分析摘要 |
| **Harness 起点** | 标注[公开信息]/[推断]/[信息不足]；禁止编造收入、利润率、客户名单 |

## 背景设定

你是某咨询公司战略部的顾问。你的团队正在研究**宇树科技（Unitree Robotics）**的商业模式，为一份机器人行业分析报告做准备。

宇树科技是中国知名的四足机器人（机器狗）和人形机器人公司，成立于2016年，总部位于杭州。公司产品包括Go系列机器狗（消费级/行业级）和H系列人形机器人。


---

## 任务目标

基于提供的模拟材料，为宇树科技生成一份完整的**商业模式画布（Business Model Canvas）**，并附商业模式总结。

## 输出要求

### 格式要求

按 BMC 9模块逐一呈现：

1. **客户细分**（Who？）
2. **价值主张**（What？）
3. **渠道通路**（How to reach？）
4. **客户关系**（How to interact？）
5. **收入来源**（How to earn？）
6. **核心资源**（What we have？）
7. **关键业务**（What we do？）
8. **重要合作**（Who helps？）
9. **成本结构**（What we spend？）

每个模块：
- 至少2个关键要点
- 可确认的标注[公开信息]
- 推断的标注[推断]
- 信息不足的标注[信息不足]
- 最后附200字商业模式总结

### 约束要求

- 只基于提供的材料
- 禁止编造具体财务数据（收入、利润率、成本）
- 禁止编造未公开披露的合作关系或客户名单
- 信息不足时明确说明，不猜测

## 评价要点

| 维度 | 重点 |
|------|------|
| **Prompt设计** | 是否要求了完整的9模块？是否有标注要求？ |
| **Context管理** | 是否综合使用了公司信息和行业分析？ |
| **Harness约束** | 是否有"禁止编造"的约束？信息不足时是否标注？ |
| **输出质量** | BMC是否完整？总结是否有洞察？ |
| **迭代过程** | 是否发现了模块间的矛盾（如收入模式vs成本结构）？ |

## 提示与技巧

### 可能的"翻车"点

1. **Agent 可能混淆模块**：把"价值主张"和"关键业务"混在一起
2. **Agent 可能编造财务数据**：如"年收入约5亿元"（材料中未提及）
3. **Agent 可能忽略信息不足**：硬把"成本结构"填满，即使材料没提

4. **如果 Prompt 没要求标注[推断]/[信息不足]**：Agent 可能把推断和事实混在一起，无法区分哪些是确定的、哪些是猜测
5. **如果只给公司信息不给行业分析**：Agent 无法判断"市场趋势"模块，BMC 会缺少外部视角
### 如何提升质量

- 明确要求"严格区分9个模块，不要合并"
- 要求"每个模块至少2个要点，但不要编造"
- 增加校验："检查是否有模块被遗漏或过度填充"

### 可扩展的方向

- 增加"商业模式对比"（与Boston Dynamics对比）
- 增加"商业模式演进"（从2016年至今的变化）
- 增加"可持续性评估"（这个商业模式能持续多久？）

汇报时建议结构：
