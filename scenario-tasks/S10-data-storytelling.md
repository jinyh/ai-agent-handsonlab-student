# S10: 数据故事化汇报

> **类型**：通用
> **难度**：★★★
> **建议时长**：90分钟
> **适合人群**：有数据分析、业务分析、战略背景

> **📌 教学材料声明**：本任务卡使用模拟数据，所有公司名称、财务数据、市场信息均为教学虚构，不构成任何投资建议。

---

## 5分钟速用框

| 项目 | 建议 |
|------|------|
| **适用场景** | 经营复盘、销售汇报、数据洞察转 PPT 大纲 |
| **推荐 Context** | `data/S10-material-A.md` 本期数据 + `data/S10-material-B.md` 去年同期数据 |
| **Harness 起点** | 只基于模拟数据；禁止编造未提及数据；同比变化异常标注[变化显著] |
| **最小成果** | 一句话结论 + 3-5条洞察 + 叙事逻辑 + 图表建议 + PPT大纲 |
| **汇报重点** | 展示 Agent 是否只是罗列数据，是否能形成“发生了什么/为什么/怎么办” |


---

## 背景设定

你是某消费品公司的数据分析经理。公司刚结束**双十一大促**，你需要向CEO汇报活动效果。

你的目标：不是罗列数据，而是**用数据讲故事**——让CEO在5分钟内理解"发生了什么、为什么、接下来怎么办"。


---

## 任务目标

基于提供的模拟数据，完成一份**数据故事化汇报**，包含：

1. **核心结论**（一句话总结）
2. **关键数据洞察**（3-5条，有数据支撑）
3. **叙事逻辑**（时间线或因果关系）
4. **可视化建议**（用什么图表呈现）
5. **PPT大纲**（10-15页）


---


## 操作提示

> 将 `data/` 目录下的 `S10-material-A.md`, `S10-material-B.md` 复制到你的项目目录中，让 Agent 读取；也可直接粘贴到 Prompt 的 Context 部分。

## 输出要求

### 格式要求

1. **核心结论**（一句话，不超过50字）

2. **关键洞察**（3-5条，每条格式：
   - 洞察标题（不超过10字）
   - 数据支撑（1-2句话）
   - 业务含义（1句话）
   ）

3. **叙事逻辑**（时间线或因果关系，用bullet points）

4. **可视化建议**（每张图表的：标题、类型、关键发现）

5. **PPT大纲**（10-15页，每页一句话说明）

### 约束要求

- 只基于提供的模拟数据
- 每条洞察必须有数据支撑
- 禁止编造未提及的数据
- 对同比变化异常（>±50%）的指标，标注[变化显著]
- 完成后列出2个数据中的"盲点"（决策需要但数据未覆盖的）

## 建议的 Prompt 框架

```
【Context】
我们是某消费品公司的数据分析经理，刚结束双十一大促。
已上传2024年双十一销售数据和2023年同期对比数据。

【Objective】
基于数据，生成一份"数据故事化"汇报，
让CEO在5分钟内理解"发生了什么、为什么、接下来怎么办"。

【Style】
数据驱动、洞察导向，每个观点有数据支撑。

【Tone】
客观、冷静，不夸大成绩，也不回避问题。

【Audience】
CEO，他熟悉业务但时间有限，需要在5分钟内抓住核心结论。

【Response】
1. 核心结论（一句话）
2. 关键洞察（3-5条，含数据支撑和业务含义）
3. 叙事逻辑（时间线或因果关系）
4. 可视化建议（图表类型+关键发现）
5. PPT大纲（10-15页）

【Constraint】
- 只基于提供的数据
- 每条洞察有数据支撑
- 不编造数据
- 变化显著的指标标注[变化显著]
- 完成后列出2个数据盲点
```

## 评价要点

| 维度 | 重点 |
|------|------|
| **Prompt设计** | 是否要求了"数据故事化"？是否有PPT大纲？ |
| **Context管理** | 是否综合使用了2024和2023年数据？ |
| **Harness约束** | 是否有"数据支撑"的约束？是否有变化显著标注？ |
| **输出质量** | 洞察是否有价值？PPT大纲是否实用？ |
| **迭代过程** | 是否发现了数据中的矛盾或异常？如何处理？ |

## 提示与技巧

### 可能的"翻车"点

1. **Agent 可能只罗列数据**：没有提炼洞察，没有叙事逻辑
2. **Agent 可能混淆因果**：把相关性当因果性（如"抖音增长快"≠"抖音效果好"）
3. **Agent 可能忽略负面信号**：只讲增长，不讲复购率下降

4. **如果 Prompt 没写 Objective 为"数据故事化"而只写"汇报"**：Agent 可能只罗列数据而不提炼洞察和叙事逻辑
5. **如果只给2024年数据不给2023年对比**：Agent 无法做同比分析，每条"增长X%"都缺少基准

### 如何提升质量

- 明确要求"不要罗列数据，要提炼洞察"
- 要求"对每个洞察，说明'这意味着什么'"
- 增加约束："必须包含至少1条负面洞察"

### 可扩展的方向

- 增加"假设分析"（如果营销费用增加20%，销售额会怎么变？）
- 增加"竞品对比"（如果知道竞品的双十一数据）
- 增加"预测"（基于当前数据，预测下季度表现）

## 汇报提示

汇报时建议结构：
1. **场景与目标**（30秒）：选了S10数据故事化，目标是向CEO汇报双十一效果
2. **Prompt 展示**（1分钟）：重点说明如何用 Objective 要求"数据故事化"而非罗列数据，如何用 Constraint 要求每条洞察有数据支撑
3. **翻车展示**（1分钟）：Agent 是否只讲增长不讲复购率下降？是否把相关性当因果性？
4. **成果展示**（1分钟）：核心结论 + 洞察列表 + PPT大纲
5. **一句话总结**（30秒）
