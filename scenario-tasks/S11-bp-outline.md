# S11: 商业计划书大纲

> **类型**：通用
> **难度**：★★★
> **建议时长**：90分钟
> **适合人群**：创业、创投、咨询背景

> **📌 教学材料声明**：本任务卡使用模拟数据，所有公司名称、财务数据、市场信息均为教学虚构，不构成任何投资建议。

---

## 5分钟速用框

| 项目 | 建议 |
|------|------|
| **适用场景** | BP 大纲、创业项目梳理、投资人沟通材料初稿 |

## 背景设定

你是一位连续创业者，正在筹备一个新的创业项目：**"AI+健康"智能营养管理**。

项目定位：基于AI大模型，为都市白领提供个性化营养管理方案（包括饮食建议、营养补充剂推荐、健康追踪）。

你的任务是：生成一份**商业计划书（BP）大纲**，用于和潜在投资人沟通。


---

## 任务目标

基于提供的项目信息，生成一份结构化的**BP大纲**，包含：

1. **执行摘要**（1页）
2. **市场机会**（痛点、市场规模、趋势）
3. **产品方案**（核心功能、技术路线、差异化）
4. **商业模式**（收入模式、定价、获客）
5. **竞争分析**（竞品、差异化、壁垒）
6. **团队**（核心成员、背景）
7. **财务预测**（收入、成本、盈利时间表）
8. **融资计划**（融资金额、用途、里程碑）

```

项目名称：NutriAI（暂定）

定位：AI驱动的个性化营养管理平台

核心功能：
1. 智能饮食记录：拍照识别食物，自动计算营养摄入

   生成个性化饮食方案
3. 营养补充剂推荐：根据饮食缺口，推荐补充剂（与品牌合作）
4. 健康追踪：整合可穿戴设备数据，追踪健康指标变化
5. AI营养师：7×24小时对话式营养咨询

目标用户：
- 一二线城市25-40岁白领
- 关注健康、有付费意愿、时间有限
- 典型场景：工作餐不知道吃什么、体检后想改善、减肥/增肌

技术路线：
- 前端：微信小程序 + iOS/Android App
- 后端：Python + 阿里云
- AI：自研营养知识图谱 + 调用GPT-4/Claude API
- 数据：用户饮食数据、健康数据、第三方可穿戴设备数据

差异化：
-  vs 薄荷健康：更智能化（AI驱动），不只是记录
-  vs Keep：聚焦营养而非运动
-  vs 医院营养师：更便宜、更便捷、更个性化

商业模式：
- 订阅制：基础版免费，高级版99元/月
- 电商佣金：推荐营养补充剂，抽成10-15%
- 企业版：面向企业健康管理，按人头收费
- 数据服务：脱敏后的营养数据（B端）

团队：
- CEO：连续创业者，前某健康App联合创始人，10年互联网经验
- CTO：前某AI公司技术总监，机器学习博士
- CMO：前某消费品牌市场总监，擅长新品牌0到1
- 医学顾问：某三甲医院营养科主任

里程碑：
- 2024年Q4：产品MVP上线，种子用户500人
- 2025年Q1：正式版上线，用户1万人
- 2025年Q2：企业版推出，合作企业10家
- 2025年Q4：用户10万人，月收入50万元
- 2026年：Series A融资，用户100万人

融资计划：
- 本轮：Pre-A轮，融资500万元
- 用途：产品研发（40%）、市场推广（30%）、团队（20%）、运营（10%）
- 里程碑：6个月内达到1万付费用户
```

## 输出要求

### 格式要求

按标准BP结构输出：

1. **执行摘要**（300字以内）
2. **市场机会**
   - 痛点（2-3条）
   - 市场规模（TAM/SAM/SOM）
   - 趋势（2-3条）
3. **产品方案**
   - 核心功能（5条，bullet points）
   - 技术架构（简要说明）
   - 差异化（vs竞品）
4. **商业模式**
   - 收入模式（4条）
   - 定价策略
   - 获客渠道
5. **竞争分析**
   - 竞品表格（功能对比）
   - 竞争优势
   - 进入壁垒
6. **团队**（核心成员，附一句话背景）
7. **财务预测**（3年表格：收入、成本、利润）
8. **融资计划**
   - 融资金额和用途
   - 里程碑

### 约束要求

- 只基于提供的项目信息
- 市场规模数据需标注来源或[估算]
- 财务预测需有假设说明
- 禁止编造团队未提及的背景
- 完成后列出2个最大的商业风险

```
- 只基于提供的项目信息
- 市场规模标注来源或[估算]
- 财务预测有假设说明

```

## 评价要点

| 维度 | 重点 |
|------|------|
| **Prompt设计** | 是否要求了完整的BP结构？是否有财务预测？ |
| **Context管理** | 是否充分利用了项目信息？ |
| **Harness约束** | 是否有市场规模来源要求？是否有风险说明？ |
| **输出质量** | BP结构是否专业？财务预测是否合理？ |
| **迭代过程** | 是否发现了商业模式中的漏洞？如何处理？ |

## 提示与技巧

### 可能的"翻车"点

1. **Agent 可能过于乐观**：市场规模估算过高，财务预测过于激进
2. **Agent 可能忽略风险**：只讲机会，不讲风险
3. **Agent 可能编造数据**：如"中国营养管理市场约1000亿元"（无来源）

4. **如果 Context 信息过载**：Agent 可能迷失在10页项目信息中，忽略最重要的差异化亮点
5. **如果只给项目基本信息不给竞品信息**：Agent 无法做竞争分析段，BP 会缺少"我 vs 别人"的关键论据
### 如何提升质量

- 要求"每个数字标注来源或[估算]"
- 要求"必须包含至少2条商业风险"
- 增加约束："财务预测需有假设说明，如'假设用户获取成本为50元/人'"

### 可扩展的方向

- 增加"投资人FAQ"（投资人最可能问的5个问题及回答）
- 增加"竞品深度分析"（选1-2个直接竞品详细对比）
- 增加"退出路径"（IPO/并购的可能性）

## 汇报提示

汇报时建议结构：
1. **场景与目标**（30秒）：选了S11 BP大纲，目标是为NutriAI生成商业计划书大纲

5. **一句话总结**（30秒）
