# S8: 竞品功能对比矩阵

> **类型**：产业/通用
> **难度**：★★
> **建议时长**：90分钟
> **适合人群**：产品、市场、战略背景，对竞品分析感兴趣

> **📌 教学材料声明**：本任务卡使用模拟数据，所有公司名称、财务数据、市场信息均为教学虚构，不构成任何投资建议。

---

## 5分钟速用框

| 项目 | 建议 |
|------|------|
| **适用场景** | 产品定位讨论、竞品功能对比、差异化策略初稿 |
| **推荐 Context** | `data/S8-material-A.md` 竞品参数 + `data/S8-material-B.md` 本公司产品定位 |
| **Harness 起点** | 只基于给定材料；未公开信息标注[未公开]；不引用传闻或用户评价 |
| **最小成果** | 对比维度 + 参数矩阵 + 综合评分 + 差异化洞察 |
| **汇报重点** | 展示评分是否有依据，是否出现不同定位产品强行比较 |


---

## 背景设定

你是某新能源汽车品牌的产品经理。公司计划推出一款**20-30万元价位的智能电动轿车**，你需要对市场上的主要竞品进行功能对比分析。

你的目标是：让管理层在10分钟内了解"我们和竞品相比，强在哪里、弱在哪里、差异化空间在哪里"。


---

## 任务目标

基于提供的模拟材料，生成一份**竞品功能对比矩阵**：

1. **对比维度确定**（价格、续航、智能化、空间、品牌等）
2. **竞品参数对比表**
3. **综合评分与排名**
4. **差异化洞察**（我们的机会在哪里？）


---


## 操作提示

## 输出要求

### 格式要求

1. **对比维度表格**（行：对比维度，列：各车型）
   - 建议维度：价格、续航、加速、智驾、座舱、空间、充电、品牌、特色

2. **综合评分表**（每维度1-5星评分，附一句话理由）

3. **排名与定位**
   - 每款车型的核心定位（一句话）
   - 每款车型的目标用户画像

4. **差异化洞察**（3条，每条≤50字）
   - 我们的机会在哪里？
   - 需要警惕的竞品是谁？

### 约束要求

- 只基于提供的材料，不补充外部信息
- 信息未公开时标注[未公开]而不是猜测
- 禁止引用无法验证的用户评价或传闻
- 完成后列出2个对比维度中信息最不充分的

## 建议的 Prompt 框架

```
【Context】
我们是某新能源汽车品牌的产品经理，计划推出20-30万元智能电动轿车。
已上传5款竞品参数和本公司产品定位。

【Objective】
基于提供的材料，生成竞品功能对比矩阵，为产品定义提供依据。

【Style】
产品经理风格，用户视角，关注"用户会怎么选"。

【Tone】
客观、务实，不贬低竞品，也不过度自信。

【Audience】
公司管理层（CEO、产品VP、营销VP），他们需要快速了解竞争格局。

【Response】
1. 对比维度表格（行：维度，列：车型）
2. 综合评分表（1-5星，附理由）
3. 排名与定位（每款车型一句话定位+用户画像）
4. 差异化洞察（3条，≤50字/条）

【Constraint】
- 只基于提供的材料
- 未公开信息标注[未公开]
- 不引用无法验证的评价
- 完成后列出2个信息最不充分的维度
```

## 评价要点

| 维度 | 重点 |
|------|------|
| **Prompt设计** | 是否要求了评分和洞察？是否定义了Audience？ |
| **Context管理** | 是否综合使用了5款竞品和本公司定位？ |
| **Harness约束** | 是否有"不引用传闻"的约束？是否有信息不充分标注？ |
| **输出质量** | 对比矩阵是否清晰？洞察是否有价值？ |
| **迭代过程** | 是否发现了参数之间的矛盾？如何处理？ |

## 提示与技巧

### 可能的"翻车"点

1. **Agent 可能把续航和加速直接对比而不标注测试标准差异**：续航标注了"CLTC"，但百公里加速的测试条件未标注
2. **Agent 可能"站队"**：过度夸赞某款车，贬低另一款
3. **Agent 可能忽略用户视角**：只比参数，不比"用户实际体验"

4. **如果 Prompt 没要求从"用户购车决策"角度分析**：Agent 可能只比参数表，忽略"用户实际会怎么选"这个关键维度
5. **如果只给竞品参数不给本公司定位**：Agent 无法分析"差异化空间"，只能做参数排名
### 如何提升质量

- 明确要求"所有参数必须标注测试标准（如CLTC）"
- 要求"从用户购车决策的角度，列出Top 3考虑因素"
- 增加约束："对比需客观，不贬低任何竞品"

### 可扩展的方向

- 增加"用户决策树"（用户会怎么选？先比什么、后比什么？）
- 增加"价格敏感性分析"（如果降价2万，竞争力如何变化？）
- 增加"时间维度"（6个月后，竞品的OTA会改变什么？）

## 汇报提示

汇报时建议结构：
1. **场景与目标**（30秒）：选了S8竞品对比，目标是对比5款20-30万智能电动轿车
2. **Prompt 展示**（1分钟）：重点说明如何用 Response 要求评分表+差异化洞察
3. **翻车展示**（1分钟）：Agent 是否"站队"夸赞某款车？是否只比参数不比用户实际体验？
4. **成果展示**（1分钟）：对比矩阵 + 差异化洞察，重点看"我们的机会在哪里"
5. **一句话总结**（30秒）
